## Gemini custom-command input

instructions. Use a non-shell file-writing tool/API to generate a new random
private file below the OS temporary `mlx-agent-gemini-command-args` root, set
mode `0600`, and write the exact opaque block. The filename and its path must
be generated by that tool, never derived from user text.

Use only an argv-list process API (not a shell string) to invoke the static
bundled executor with this exact shape:

`PYTHONPATH=<skill-dir>/src python3 -m mlx_agent.gemini_executor --capability adopt --args-file <generated-private-path>`

Replace `<skill-dir>` with this skill directory and `<generated-private-path>`
only with the file API's generated path. The executor validates, consumes, and
removes the file. Never interpolate raw command text into a shell string. If a
non-shell file-writing tool/API or argv-list process API is unavailable, stop
and request the documented arguments conversationally one field at a time.

Use the durable adoption state owned by the structured CLI. Start with a user-visible state path and requested roles:

`python3 <skill-dir>/scripts/mlx-agent adopt start --state <state-path> --role <role> --json`

If the state already exists or an earlier run was interrupted, continue it with:

`python3 <skill-dir>/scripts/mlx-agent adopt resume --state <state-path> --json`

Report the CLI state and recommendations. Do not recreate adoption policy in this adapter. This operation must not download model weights or change configuration; any later download or mutation requires explicit user confirmation and the reviewed CLI preview.
